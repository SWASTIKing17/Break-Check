# Conversation Brief: SubMachine Logic Hardening v2.8.7
**Date**: 2026-04-25

## Summary of the Conversation
The user requested technical implementation of v2.8 logic fixes for the SubMachine Premiere Pro extension. The goals were to harden timeline logic to ensure a flawless experience when editing subtitle phrases. We accomplished this through three main pillars:
1. **Gap-Filling Surgery**: Ensuring contiguous, word-to-word gap removal within a merged or modified phrase.
2. **3-Track Staircase & 3-Color Soft Palette**: To achieve clear visual differentiation of adjacent phrases on the timeline, we enforced a 3-track staircase rule and mapped phrases to a 3-color soft palette (`[1, 3, 4]`).
3. **Typo Sync & Style Merge**: Upgrading the 'Sync Text & Typeface' tools so they broadcast style changes across all selected phrases, but preserve the unique text of the non-master phrases.

## Decisions Made
- **Color Choice**: Soft colors [1, 3, 4] (Cerulean, Lavender, Caribbean) were chosen.
- **Track Staircasing**: Instead of returning modified clips to the master's original track, the system mathematically calculates an available video track out of tracks 1, 2, and 3 that does not clash with the tracks occupied by the outer neighbor phrases.
- **`!inMasterPhrase` Text Protection**: We aligned both `syncText` and `syncAll` to determine if a selected clip belongs to the "master phrase". If not, text is protected while font styles and transformations are synchronized.
- **ExtendScript Limitations**: Because moving a clip between tracks via ExtendScript resets its MOGRT parameters to default, we extract and re-apply a `masterStyleMap` containing all text, style, progression, and motion data whenever clips change tracks.

## Code Changes
1. **`mogrt.jsx`**: Enabled dynamic allocation of a third video track when creating new MOGRTs. Soft color rotation was added to `createCaptions`.
2. **`phrasing.js`**: Updated the loop generating sequences to cycle through tracks 1, 2, and 3 instead of using a modulo-2 calculation.
3. **`sync.jsx`**: Transformed `syncAll` to utilize the same `phraseIndex` protection logic as `syncText`.
4. **`timeline.jsx`**: Integrated the `getSafeAlternatingTrack` calculation into the Split, Join, and Split & Join tools. Added `_getClipTrackIndex` helper to deduce active clip tracks accurately.
