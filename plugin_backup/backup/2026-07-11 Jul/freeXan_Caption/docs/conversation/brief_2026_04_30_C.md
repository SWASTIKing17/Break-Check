# Conversation Brief - Session 2026-04-30-C

## Context
The Edit tab (React Command Center) was permanently visible and covering all other tabs (Workflow, Tools, etc.) in the SubMachine Premiere Pro extension.

## Root Cause (3-Part CSS Conflict)
1. **Wrong Class**: The Edit tab's content div used `.tab__content_react` instead of the standard `.tab__content`, bypassing the legacy radio-button hide/show system.
2. **Force Visible**: The custom class had `display: block !important` and `position: absolute` with `z-index: 99`, making it permanently visible.
3. **Grid on Wrapper**: `command_center.css` applied a grid layout to `#commandCenterTab` (the `.tab` wrapper div) instead of `#react-edit-root` (the inner React content), breaking the wrapper's `display: inline-block` needed by the tab system.

## Fix Applied
- Changed Edit tab content div from `.tab__content_react` → `.tab__content`
- Rewrote inline `<style>` to use `position: absolute` + `opacity: 0/1` + `pointer-events` system compatible with all tabs
- Moved grid layout from `#commandCenterTab` → `#react-edit-root` in `command_center.css`
- Updated responsive media query to match

## Files Modified
- `panel/panel.html` — Inline `<style>` block rewritten, Edit tab class changed
- `panel/command_center.css` — Grid layout selector changed, media query updated
- `docs/logs/changelog.md` — v3.5.5 entry added

## Guidance for Next Agent
- The tab system is pure CSS (radio buttons + sibling selectors). No JS controls tab visibility.
- All `.tab__content` divs use `position: absolute; opacity: 0` by default, shown via `.myTab:checked ~ label ~ .tab__content { opacity: 1 }`
- The Edit tab's checked state adds `display: flex` for the React grid layout
- All features from previous conversations (Word Surgery, Selection Injection, Sync) remain intact
