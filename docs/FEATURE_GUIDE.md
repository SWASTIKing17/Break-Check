# freeXan — User-Facing Feature Guide

**App:** freeXan by BloomX
**Version:** v2.5.2
**Document purpose:** Every feature, behavior, and interaction the user can observe — derived directly from reading the code, function by function. Use this to verify each item is intentional behavior (feature) vs. an unintended outcome (bug).

**How to read this doc:**
- Items are grouped by surface area (Tray, Main Window, Overlay, CEP Panel, Background).
- Each item is numbered with a stable ID like `F-MAIN-001` so you can call out specific items.
- For each item: **Behavior** (what the user sees) → **How to trigger** (steps) → **Source** (file:line).

## 11. Linked Folders ↔ Premiere Bins (v3.2 → v3.4)

## 12. Halo Picker on the Overlay (v3.4)

A drop-time picker that lets the user choose which linked folder receives a file. Triggered by holding Ctrl while dropping on the overlay pill. Replaces the v3.3 "hold key while dropping" gesture, which was architecturally unworkable on Windows because keyboard focus stays on the drag source (File Explorer) for the whole drag.

### 12.1 Trigger Model

| ID | Modifier on drop | Behavior | Source |
|---|---|---|---|
| F-HALO-001 | (none) | Standard slot mapping path. Unchanged from prior versions. | `renderer/overlay.js` drop handler |
| F-HALO-002 | **Ctrl** | Halo picker opens; file is staged for COPY after user selection. | `renderer/overlay.js` `openHaloPicker(files, moveMode=false)` |
| F-HALO-003 | **Ctrl + Shift** | Halo picker opens; small `MOVE` chip is shown below the picker; file is staged for OS-level MOVE after user selection. | `renderer/overlay.js` `openHaloPicker(files, moveMode=true)` |
| F-HALO-004 | **Shift** alone | Standard slot mapping path but with `moveSource: true` — the file is moved instead of copied. No picker. | `renderer/overlay.js` drop handler |
| F-HALO-005 | Modifier source | Modifier state is captured by `preload.js` from the `DragEvent` and forwarded as a second argument to `onFilesDropped(filePaths, { ctrlKey, shiftKey, altKey })`. The drag event carries modifier state regardless of which window has keyboard focus, which is why this works where the v3.3 keydown listener didn't. | `preload.js` window drop listener |
| F-HALO-006 | **preventDefault is mandatory** | The window-level drop handler in `preload.js` MUST call `e.preventDefault()` and `e.stopPropagation()`. Without these, Electron's default behaviour navigates the renderer to the dropped file's `file://` URL, destroying the overlay HTML (was the root cause of "bubble not clickable" + "white window" reports in v3.4.0). | `preload.js:11-14` |
