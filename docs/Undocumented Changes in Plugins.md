This JSX function lives in sync.jsx and is identical in both backup and CEPs. However sync.jsx has a key change in how color data arrays are built for MOGRT properties: - OLD format: [r, g, b, a] - NEW format: [a, r, g, b] with added index: k field  This is NOT the direct issue. The actual root cause is:  getTimelinePhraseMap() is wrapped in safeCall() which returns a JSON string {ok: boolean, data: ...} envelope. The EditView.tsx calls execute('getTimelinePhraseMap') via useJsx hook, which calls csInterface.evalScript. The React useJsx hook must unwrap the safeCall envelope. If it is NOT doing this correctly, the  esponse in:  const fresh = Array.isArray(response) ? response : [];  ...will NEVER be an Array  it will be the envelope object {ok: true, data: [...]} or the raw JSON string. This means  imelineMap stays [] and the UI shows EmptyDashboard.  ### Issue 2: Connect Button State / wsConnected  The useFreeXanWs.ts hook sends {type: 'getprojectstate'} on WebSocket open. The server (main.js:582) handles this and responds with {type: 'projectstate', ...}. The hook correctly sets wsConnected = true on onopen. If the Electron app is not running, connection fails silently and no word bubbles appear.  ### Issue 3: LinkfreeXan Not Working  The LinkfreeXan/ext.js no longer automatically calls  equestAudioLibrary() after WebSocket connect. In the backup, the ext auto-requested audio data on connection, populating the UI. In the current version, this auto-request was removed. Without an explicit trigger, the audio list never loads.

### Issue 4: ExtendScript Engine Crash (EvalScript error)
**Root Cause:** The reeXan Caption extension was failing to initialize completely in Premiere Pro, returning EvalScript error and reporting a disconnected engine state. The actual bugs were in the ExtendScript files:
1. core/mogrt.jsx: Syntax error on /[^/.]+/. ExtendScript ES3 misparses unescaped forward slashes inside brackets. Escaped to /[^\/.]+/.
2. core/timeline.jsx: Syntax error Illegal use of reserved word 'char'. A variable named char was used. Renamed to cChar.
**Fix:** Modified these backend .jsx files in CEPs/freeXan_Caption and ran Install_freeXan_Caption.bat to re-deploy.


### Issue 5: Text Formatting and SRT Export
**Root Cause:** The sm_tools_format_text_selected_mogrts function was strictly checking for exactly '? Text Input' or 'Text Input', skipping MOGRTs using 'Text' or 'Source Text', which caused the 'No clips updated' error in the UI. In addition, the user requested the ability to export Word-by-Word and Phrased SRT files from the EditView.tsx timeline map.
**Fix:** Modified core/timeline.jsx to loop over all text param permutations. Added exportUtils.ts to panel-src/src/lib with generateWbwSrt and generatePhrasedSrt utilities, and injected export buttons into the EditView.tsx footer.

### Issue 6: Lengthy Add/Remove Word Execution
**Root Cause:** Adding or removing words re-injected Master Style across all clips in a phrase and performed slow O(N) timeline scanning.
**Fix:** Replaced with duplicate/trim logic in 	imeline.jsx (sm_tools_add_word_v28, sm_tools_remove_word_v28). Added sm_tools_reset_progression_v28 and UI trigger in WordEditGroup.tsx.
